# CS-370-Current-Emerging-Trends
*Module Eight Journal

On this project, I built a pirate intelligent agent using deep Q-learning. Initially, I received maze environment, assistance files, and neural network setup code. I was responsible for finishing the Q-training algorithm, including exploration vs. exploitation, experience storage, replay data training, win rate monitoring, and agent evaluation utilizing the completion check.

My Codio program wouldn't finish. The code was error-free, however the environment kept shutting down before training could complete. I was able to test and examine the training process, but I couldn't always obtain a full convergence run. I learned that AI training may be resource-intensive and that code and environment can both be to blame.

I studied how reinforcement learning and neural networks relate to computer science in this course. Computer scientists break down issues, test multiple ways, and improve solutions over time. This experiment taught me that AI doesn't instantly "know" the solution. Data, incentives, trial, and error, and persistent training teach it.

Because I now realize that AI creation requires testing and adjusting, my problem-solving has altered. Epsilon decay, epochs, and training parameters impact agent learning. I also realized that accurate, secure, and transparent AI systems need ethical actions. When making data- or model-based choices, developers must examine how outcomes influence people and organizations.
